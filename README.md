# GAOGAO 発注ポータル（GitHub Pages × Googleスプレッドシート）

いまお使いの**Googleスプレッドシートをそのまま読み込んで**、見やすいWebページに表示します。
APIキーは不要。**更新する人はスプレッドシートを編集するだけ。GitHubは最初の1回だけ。**

## ページ構成（発注ポータル）
| ページ | 役割 | データ |
|---|---|---|
| `index.html` | 供給スケジュール・価格表・カート発注 | 供給/価格シートを読む |
| `order-confirm.html` | 注文確認（顧客が自分の注文を確認・受付中は修正/キャンセル） | 「注文」シートを読む＋Apps Scriptで更新 |
| `admin.html` | 承認・受付管理（GAOGAO用・URL限定） | 「注文」を読む＋Apps Scriptで承認/単価/状態更新 |
| `jisseki.html` | 納品実績（品目別・会社別・月別の集計、グラフ、CSV） | 「注文」を読む |

- 読み取りは全ページ APIキー不要（gviz）。書き込み（承認・修正）だけ Apps Script を使用。
- 注文の状態：**受付 → 承認済み → 納品済み**（／キャンセル）。
- 承認・実績ページは認証なし（**URLを知っている社内だけで運用**）。共有範囲にご注意ください。

### 設定する場所（Apps Script接続後）
Apps Scriptの `/exec` URL を **2か所**に貼ります：
1. `index.html` の `CONFIG.ORDER_WEBAPP_URL`（カート送信用）
2. `assets/orders.js` の `CONFIG.ORDER_WEBAPP_URL`（確認・承認・実績の更新用）

※ 未設定のうちは、発注はメール送信／承認・修正は「画面表示のみ（保存されない）」のデモ動作になります。
Apps Scriptは管理機能対応版（`apps-script.gs`）を使ってください（「注文」タブを自動作成し、受付/承認/単価/状態を扱います）。

```
Googleスプレッドシート（今のシートのまま・みんなで編集）
        │  ページを開くたびに自動取得（APIキー不要）
        ▼
   GitHub Pages（index.html）
        │
        ▼
   SRBC・クオリア（見るだけ・URLで共有）
```

このHPは元の3シートのうち **「推定供給（月間）」** と **「圃場別価格表」** を読み、
- 週間（kg/週）は「月間 ÷ 3.5」で**自動計算**
- 出荷ステータス（出荷可能／来週／◯月◯日頃から）は**見た日の日付で自動判定**
します。スプレッドシート側のレイアウトは今のままでOKです。

## 発注機能（カート → スプレッドシート記録 ＋ メール通知）
供給スケジュール表の右端「発注(kg)」に数量を入れると、右下のカートに入ります。
カートで会社・担当者・希望納品日を入れて送信すると、注文がスプレッドシートの
**「注文」タブに自動記録**され、**GAOGAOにメール通知**が届きます（金額は後日GAOGAOが確定）。

### 接続前でも使えます
`index.html` の `ORDER_WEBAPP_URL` が空のうちは、注文ボタンは**メール下書き(mailto)**を開きます。
下記でApps Scriptを接続すると、自動記録＋通知に切り替わります。

### Apps Script の接続手順（初回のみ・5〜10分）
1. 注文を記録したいGoogleスプレッドシート（今の供給シートでOK）を開く
2. **拡張機能 → Apps Script** を開く
3. `apps-script.gs` の中身を全部貼り付け、`NOTIFY_EMAIL` を通知先アドレスに変更して保存
4. **デプロイ → 新しいデプロイ → 種類「ウェブアプリ」**
   - 「次のユーザーとして実行」= **自分**
   - 「アクセスできるユーザー」= **全員**
   - → デプロイ（初回は権限の承認を求められます → 許可）
5. 表示された **ウェブアプリURL（末尾 `/exec`）** をコピー
6. `index.html` の `CONFIG.ORDER_WEBAPP_URL` にそのURLを貼って保存 → GitHubに反映
7. 動作確認：HPでカートに入れて送信 → 「注文」タブに行が増え、メールが届けば成功

> ※ スプレッドシートを編集（注文記録）するのはApps Scriptで、これは**あなた（所有者）として実行**されます。
> だから閲覧者は中身を書き換えられないのに、注文だけは記録できます。

## 設定（`index.html` 上部）
すでに今回のスプレッドシート用に設定済みです。差し替えるときだけ編集します。
```js
const CONFIG = {
  SHEET_ID: "1hyNpkwXwF5JQE3pLY3DFhGGEX5228tQMbnBsyar7Ylg", // スプレッドシートのID
  SUPPLY_SHEET: "推定供給（月間）",   // 供給のタブ名
  PRICE_SHEET:  "圃場別価格表",        // 価格のタブ名
  WEEKS_PER_MONTH: 3.5,
  ORDER_WEBAPP_URL: "",               // Apps Scriptの /exec URL（空ならメール送信）
  ORDER_EMAIL: "toruhyuga0810@gmail.com", // mailtoフォールバック先
  COMPANIES: ["SRBC", "クオリア"]     // 注文者の選択肢
};
```
※ スプレッドシートは「共有 → リンクを知っている全員 → 閲覧者」にしておくこと（編集権は社内だけに付与）。

## ローカルで動作確認する
`index.html` を**ダブルクリックで開くとデータが出ません**（ブラウザのCORS制限のため）。
簡易サーバー経由で開いてください。ターミナルでこのフォルダに入り：
```
cd gaogao-hp
python3 -m http.server 8000
```
ブラウザで `http://localhost:8000` を開く → 表が出れば成功。
（公開後の GitHub Pages では制限なく動きます。）

## GitHub Pages で公開する（初回のみ）
1. GitHub で新規リポジトリ作成（例：`gaogao-data`、Public）
2. このフォルダの `index.html` をアップロード（ドラッグ&ドロップでOK）
3. Settings → Pages → Source を **Deploy from a branch** / Branch **main /(root)** → 保存
4. 数分後 `https://（あなたのID）.github.io/gaogao-data/` で公開 → URLを共有

## 運用（2回目以降）
- 数量・価格・納品開始日を変えたい → **スプレッドシートを編集するだけ**。
  HPは開き直す（または「↻ 最新に更新」ボタン）で反映されます。
  ※ Google側キャッシュで数分遅れることがあります。
- 品目の追加・削除もスプレッドシートでOK（HPが自動で行を増減）。

## 入力のちょっとした注意（今のシートのまま）
- 供給「推定供給（月間）」：数量はkgの数字。立ち上がり・端境期は `◯`、供給なしは空欄。
  納品開始日は `2026/8/1` のような日付。
- 価格「圃場別価格表」：各圃場ブロックの「卸値/kg」に数字のみ。取扱なし／未定は空欄。
  圃場見出し（`第1圃場：…`）の位置はそのまま維持してください（HPがこれを目印に読みます）。

## 別の資料も増やしたくなったら
同じリポジトリにフォルダを足して管理できます：
```
gaogao-data/
├── index.html              ← この供給スケジュール
└── （例）member-list/       ← 次の資料はフォルダ追加 → 目次トップから各ページへリンク
```
